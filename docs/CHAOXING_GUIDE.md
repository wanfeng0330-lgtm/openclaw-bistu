# 超星学习通接入指南

## 概述

超星学习通（Chaoxing）是北信科使用的在线学习平台，提供课程管理、作业提交、在线签到等功能。

## 接入方式

### 方式一：手机号 + 密码登录

最简单的方式，在 `config.json` 中配置：

```json
{
  "chaoxing": {
    "base_url": "https://mooc1-1.chaoxing.com",
    "phone": "你的手机号",
    "password": "你的密码"
  }
}
```

### 方式二：Cookie 登录

如果手机号密码方式不可用，可以手动获取 Cookie：

1. 在浏览器打开 https://mooc1-1.chaoxing.com
2. 登录学习通
3. 打开浏览器开发者工具 (F12)
4. 在 Network 面板找到请求，复制 Cookie 头
5. 将 Cookie 写入配置文件

### 方式三：学习通 API 代理

学习通的移动端 API 相对稳定，本 Skill 优先使用移动端接口：

- 基础 URL: `https://mooc1-api.chaoxing.com`
- 认证方式: Cookie（登录后自动获取）

## 可用功能

| 功能 | API 端点 | 说明 |
|------|----------|------|
| 课程列表 | `/mycourse/backclazzdata` | 获取当前学期课程 |
| 作业查询 | `/mycourse/workindex` | 获取未完成作业 |
| 签到查询 | `/mycourse/signindex` | 获取待签到列表 |
| 课程资源 | `/mycourse/stuindex` | 获取课程资料 |

## 签到功能说明

⚠️ **重要提示**：学习通签到功能**仅提供提醒和状态查询**，实际签到需要：

1. **普通签到**：在手机端点击签到按钮
2. **二维码签到**：扫描教师展示的二维码
3. **手势签到**：在手机端输入正确的手势
4. **位置签到**：在指定位置范围内签到

本 Skill 不支持自动签到，原因：
- 违反学习通使用规定
- 可能导致账号被限制
- 影响教学秩序

## 学习通 API 逆向说明

学习通没有公开 API 文档，本 Skill 的接口是通过以下方式获取：

1. 使用 mitmproxy 抓取学习通 App 的请求
2. 分析请求参数和响应格式
3. 在 Python 中模拟请求

如果接口变更，可能需要重新抓包分析。

## 常见问题

### Q: 登录失败

A: 检查：
- 手机号和密码是否正确
- 是否需要图形验证码
- 账号是否被限制

### Q: 作业列表为空

A: 可能原因：
- 当前学期没有课程
- 课程尚未发布作业
- Cookie 已过期，需要重新登录

### Q: 签到提醒不及时

A: 建议配合 OpenClaw 的定时任务功能，定期检查签到状态。
