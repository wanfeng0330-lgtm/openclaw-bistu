# 统一身份认证 (SSO) 登录说明

## 概述

北京信息科技大学使用统一身份认证平台 (`sso.bistu.edu.cn`)，所有校园系统（教管平台、一网通办等）都通过 SSO 登录。

## 支持的登录方式

### 1. 账号密码登录（推荐）

- 用户名：学号或教工号
- 密码：初始密码通常为身份证后六位，建议修改

### 2. 扫码登录

- 打开 SSO 登录页面，切换到扫码模式
- 使用微信扫描二维码
- 在手机端确认登录

### 3. 短信验证码登录

- 输入绑定的手机号
- 获取并输入短信验证码

### 4. WebAuthn 登录

- 支持指纹、面部识别或安全密钥
- 需要先在用户中心注册 WebAuthn 凭证

## 本 Skill 的 SSO 接入方式

openclaw-bistu 通过模拟 SSO 登录流程获取认证 Cookie：

1. 访问 SSO 登录页面，获取 `lt` 和 `execution` 参数
2. 提交用户名、密码等登录表单
3. 获取认证后的 Cookie（包含 `route`、`JSESSIONID` 等）
4. 使用 Cookie 访问教管平台和一网通办

## Cookie 有效期

- SSO Cookie 通常有效期为 **7 天**（如果勾选"7天内免登录"）
- 过期后需要重新登录
- 本 Skill 的脚本会自动处理 Cookie 过期和重新登录

## 安全建议

1. **不要将 config.json 分享给他人**
2. **定期修改 SSO 密码**
3. **如发现异常登录，立即修改密码**
4. **使用最小权限原则**，只配置需要的服务

## 常见问题

### Q: 登录失败提示"需要验证码"

A: 连续登录失败会触发腾讯验证码，建议：
- 确认用户名和密码正确
- 手动在浏览器登录一次清除验证码状态
- 尝试使用扫码登录方式

### Q: Cookie 过期频繁

A: 可以尝试：
- 在 SSO 登录时勾选"7天内免登录"
- 使用扫码登录方式，Cookie 有效期更长

### Q: 忘记密码怎么办

A: 
- 访问 https://sso.bistu.edu.cn 点击"重置密码"
- 或联系学校信息中心
